## Learned User Preferences

- Always communicate with the user in German (UI copy, explanations, and coach-facing text).
- **Git (Dev):** Push changes directly to `master` — no feature branches or separate PRs unless the user asks otherwise.
- Onboarding: Ziel single-select; Hindernis multi-select; order branche → … → rolle (mid-flow) → … → vorname + firmenname (steps 9–10, before auth).
- Phase 1 coach: no „typischer Tag“; offer type then Projektablauf (customer acquisition → delivery); company facts → canvas + session memory; intro → „Lass uns gleich starten:“ → first diagnostic question in message 2; reference onboarding; one question per message, max one Zwischenfrage before next script step.
- „Rundherum“ questions (Verwaltung, Buchhaltung, Wissen, etc.) for all team sizes including Solo.
- Chat coach: plain text on white; no `---`, dialog prefixes, or meta filler; no markdown tables (numbered lists for comparisons); client keeps raw `<options>` in message state, DB/store gets stripped text; image attachments = thumbnail preview only (no filename/OCR badge); attachment metadata hidden via `<user_attachments>` — no OCR or filenames in visible bubbles; voice mic beside send (Voxtral), recording status must not shift input layout.
- Landing page (marketing): customer-facing German; light `bg-slate-50` + `bg-grid` (no dark hero gradients); no scroll phase/roadmap connector line; story Verstehen → Planen → Bauen → Live-Schalten; avoid deployen, Canvas, Workflow-Engine, n8n, Human-in-the-loop on LP — use Plan/Roadmap/Umsetzen/Live-Schalten.
- Phase transition: manual button after summarize; phases 2+ via hidden init from `lib/phase-welcome.ts` / `lib/session-kickoff.ts` — never Phase 1 onboarding intro on phase switch or empty session.
- Phase 2–3: Phase 2 recap + A/B/C once, tools per pain, implicit automation hints, Umsetzer at end, use_cases = status quo only; Phase 3 gap-fill, no re-ask A/B/C, update existing workflows (not new), human-in-the-loop before posting/strategic steps.
- Phase 4 coach + canvas: no finished workflows at start — ask which to build first, then live workflow tool-call build; coach knows workflow editor UI; no re-interview, A/B/C, or tool re-discovery; deploy cards at Phase‑4 node only; ~90% modal with margin; React Flow; fit-view on workflow open; custom minimap styled like canvas nodes (not blue boxes); trailing `+` connector after terminal nodes (n8n-style); nodes icon-only with label below, drag follows cursor; n8n catalog SSOT (categorized picker, auto-pick on create, MCP `search_nodes` supplements search); real tool icons; red credential borders; Google/Microsoft OAuth = inline „Mit … verbinden“ (no credential dropdown/popup/API-key for OAuth2 types); Deploy in modal only; rich per-node config; per-node German purpose note (not generic EN node description); AI Agent/summarization chain sub-node ports (model/memory/tools); bottom-center editor chat as Setup-Coach — same coach context as main chat, output stays in editor only (configure nodes, concrete webhook/path next steps from Phase 3); AI edits preserve edges + full workflow overview; large popups for node switch/add; first node must be trigger.
- Account overview in app sidebar shows email, name, and auth provider.
- Never refactor `RoadmapCanvas` curved path, phase checkpoints, or side PhaseNode TitleBox — only content rails, card placement, and doc-first dedup.
- Canvas/memory blobs only facts from the conversation — no speculation or invented fields; terminal logs use `[agent-sync][canvas|memory][event]`.
